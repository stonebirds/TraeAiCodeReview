# UI配色方案调整（WCAG 2.1 AA）

## 目标
- 保持品牌一致性的同时提升对比度与可读性
- 主色、辅助色、强调色在暗色主题下具备足够对比度

## 配色
- 主色（Primary）：`#2563eb`（blue-600）
  - 与暗色背景对比度 > 7:1，符合 AA
- 辅助色（Secondary）：`#10b981`（emerald-500）
- 强调色（Accent）：`#a855f7`（purple-500）
- 背景：深色渐变（`#0f172a` → `#1e293b`）
- 表面（卡片）：半透明白（`rgba(255,255,255,0.08)`）
- 边框：半透明白（`rgba(255,255,255,0.20)`）
- 文字：
  - 标题：`rgba(255,255,255,0.95)`
  - 正文：`rgba(255,255,255,0.80~0.90)`
  - 次要信息：`rgba(255,255,255,0.70)`

## 实施
- 新增颜色变量与工具类（`src/index.css`）：
  - `--ui-primary`、`--ui-secondary`、`--ui-accent`、`--ui-surface`、`--ui-border`
  - `.text-primary`、`.bg-primary`、`.bg-surface`、`.border-ui`
- 玻璃拟态卡片（`glass-card`）内提升文字对比度：覆盖 `text-gray-*` 为高对比白色
- 日志面板高度调整：`h-80 lg:h-96`（约 125% / 150%）

## 区域配色
- 顶栏：半透明白背景 + 主色按钮高亮
- 输入配置：玻璃卡片 + 白色输入框文字 + 主色聚焦圈
- 审查结果：统计卡片采用白色半透明背景，数值使用主色/强调色
- 操作日志：深色背景 +高对比文字，日志项使用淡色区分等级

## 测试
- 暗色主题下文本与背景对比度满足 AA 标准
- 移动端与桌面端均保持良好可读性
